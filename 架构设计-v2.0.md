## 一、实现方案 + 框架选型

### 技术栈（延续现有）
- **后端**：Node.js + Express + better-sqlite3（不变）
- **前端**：Vue3 + Element Plus（不变）
- **红色主题**：#E53535（不变）

### 架构原则
1. **向后兼容**：现有数据不迁移，新增表与现有表共存
2. **配置化优先**：成本科目、核算模式等通过配置表管理，不硬编码
3. **标准合规**：数据库字段命名对齐GB/T 46210术语
4. **渐进式升级**：P0需求先上，P1/P2后续版本迭代

## 三、数据结构和接口（类图）

### 核心数据表设计

#### 1. cost_accounts（成本科目表）
```sql
CREATE TABLE cost_accounts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    account_code VARCHAR(20) NOT NULL UNIQUE,  -- 科目编码（如1001）
    account_name VARCHAR(100) NOT NULL,         -- 科目名称
    account_type VARCHAR(20) NOT NULL,           -- 科目类型（direct_material/fuel_labor/direct_labor/manufacturing_overhead）
    parent_id INTEGER,                           -- 父科目ID（支持多级）
    is_standard BOOLEAN DEFAULT 0,              -- 是否标准科目（财会〔2013〕17号）
    is_enabled BOOLEAN DEFAULT 1,               -- 是否启用
    tenant_id VARCHAR(50),                       -- 租户ID（SaaS模式）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. cost_estimates（成本估算表）
```sql
CREATE TABLE cost_estimates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    estimate_no VARCHAR(50) NOT NULL UNIQUE,     -- 估算编号
    project_name VARCHAR(200) NOT NULL,          -- 项目名称
    estimate_method VARCHAR(20) NOT NULL,        -- 估算方法（analogy/parametric/bottom_up）
    estimated_cost DECIMAL(15,2),                -- 估算成本
    accuracy_rate DECIMAL(5,2),                  -- 精度（±百分比）
    estimate_date DATE NOT NULL,                 -- 估算日期
    estimator_id VARCHAR(50),                    -- 估算人ID
    status VARCHAR(20) DEFAULT 'draft',          -- 状态（draft/submitted/approved）
    tenant_id VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 3. cost_budgetary（成本概算表）
```sql
CREATE TABLE cost_budgetary (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    budgetary_no VARCHAR(50) NOT NULL UNIQUE,    -- 概算编号
    project_name VARCHAR(200) NOT NULL,          -- 项目名称
    feasibility_stage VARCHAR(50),                -- 可行性研究阶段
    total_investment DECIMAL(15,2),              -- 总投资概算
    construction_cost DECIMAL(15,2),             -- 工程建设费
    equipment_cost DECIMAL(15,2),                -- 设备购置费
    other_cost DECIMAL(15,2),                    -- 其他费用
    approval_status VARCHAR(20) DEFAULT 'pending',  -- 审批状态
    approver_id VARCHAR(50),                     -- 审批人ID
    approved_at DATETIME,                        -- 审批时间
    tenant_id VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4. quality_cost_standard（质量成本标准分类表）
```sql
CREATE TABLE quality_cost_standard (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cost_category VARCHAR(20) NOT NULL,          -- 成本类别（prevention/appraisal/internal_failure/external_failure）
    cost_item VARCHAR(100) NOT NULL,             -- 费用项目（如：质量培训费）
    account_code VARCHAR(20),                     -- 对应会计科目
    is_enabled BOOLEAN DEFAULT 1,
    tenant_id VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 七、共享知识（跨文件约定）

1. **术语规范**：所有变量名、数据库字段名使用下划线命名法，对齐GB/T 46210英文术语
2. **租户隔离**：所有新表均需包含tenant_id字段（SaaS模式）
3. **审计日志**：所有配置表（cost_accounts等）的变更需记录到audit_logs表
4. **API响应格式**：统一使用`{ code: 200, data: ..., message: '...' }`格式
